# ShopDB — консольная база данных продукции с гибридным шифрованием

Учебный проект (курсовая работа, 2 семестр, дисциплина «Языки программирования», РТУ МИРЭА).
Консольное приложение на C++ для ведения базы данных товаров магазина с автоматическим шифрованием файла данных при выходе.

## Возможности

- Хранение разнородной продукции через полиморфизм (базовый класс `Product`, наследник `Book`)
- Добавление, удаление и редактирование записей по номенклатурному номеру
- Просмотр всей базы и выборки:
  - список продукции определённого вида
  - список книг определённого автора
  - суммарная стоимость продукции определённого вида
- Загрузка/сохранение базы в файл
- **Гибридное шифрование файла данных при выходе из программы**

## Архитектура

| Файл | Назначение |
|------|-----------|
| `main.cpp` | Точка входа, меню, обработка команд пользователя |
| `product.h` | Базовый класс `Product` (id, вид, цена, количество) |
| `book.h` | Класс `Book : public Product` (автор, название, год, переплёт) |
| `shopdb.h` / `shopdb.cpp` | Класс `ShopDB` — хранение, CRUD, выборки, save/load |
| `template.h` | Шаблонный ввод `read_typename<T>()` с валидацией через `parse<T>()` |
| `EN_DEcryption.h` | Функции шифрования/дешифрования файла базы |

Использованы: наследование и виртуальные методы (`virtual void print()`), шаблоны с частичной специализацией для строк, контейнеры STL (`vector<Product*>`), RAII (шифрование вызывается контролируемо при выходе).

## Схема шифрования

Реализована гибридная схема поверх OpenSSL:

1. При выходе генерируется случайный 64-символьный пароль.
2. Файл базы шифруется симметрично: **AES-256-CBC** (`-iter 200000`).
3. Пароль шифруется асимметрично **RSA** на открытом ключе (`rsa.public`) → `key.txt.enc`.
4. При следующем запуске приватным ключом (`rsa.private`) расшифровывается пароль, затем — сам файл базы.

Таким образом на диске в открытом виде не остаётся ни данных, ни ключа.

## Сборка

Проект собирается в Qt Creator (qmake), тулчейн **Qt 5.14.2 MinGW 32-bit**:

```
qmake WorkCourseW.pro
mingw32-make
```

Для работы шифрования рядом с исполняемым файлом должна лежать папка `OpenSSL-Win64\bin\` с `openssl.exe`, а также пара ключей `rsa.public` / `rsa.private`.

## Запуск

```
WorkCourseW.exe <имя_файла_базы>
```

Программа проверит наличие зашифрованного файла, при необходимости расшифрует его и загрузит базу, после чего откроет меню.

## Известные ограничения

Это учебный проект, поэтому ряд решений упрощён и не предназначен для боевого использования:

- Пароль генерируется через `rand()` с сидом от `time()` — это не криптостойкий ГПСЧ; для реальной задачи нужен CSPRNG (например, `RAND_bytes` из OpenSSL).
- Шифрование выполняется через вызов внешнего `openssl.exe` (`system()`), а не через libcrypto — сделано для наглядности.
- Приватный ключ `rsa.private` в репозиторий **не** коммитится (см. `.gitignore`).
